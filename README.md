# LogLint

**LogLint** — это статический анализатор (линтер) для языка Go, предназначенный для проверки стиля, безопасности и качества сообщений в логах. Он поддерживает стандартные пакеты `log`, `slog`, а также популярные библиотеки `zap` и `logger`.

## Основные возможности

* **Контроль регистра** — проверка того, что сообщения логов начинаются со строчной буквы (согласно *Go Code Review Comments*).
* **Локализация** — запрет использования не-ASCII символов (кириллицы и др.) в логах.
* **Безопасность** — обнаружение и замена чувствительных данных (пароли, токены, API-ключи) на безопасные заглушки.
* **Чистота текста** — ограничение использования специальных символов с возможностью настройки исключений.
* **Гибкая настройка** — исключение файлов по регулярным выражениям и выбор целевых методов логирования через YAML.
* **Auto-fix** — поддержка автоматического исправления найденных нарушений.

## Установка

### Установка как консольной команды

Вы можете установить линтер напрямую в `$GOPATH/bin` и использовать его как обычную команду:

```bash
go install github.com/freekopik/loglint/cmd/loglint@latest
```

### Сборка из исходного кода

Или соберите бинарный файл вручную из корня проекта:

```bash
go build -o loglint ./cmd/loglint
```

## Конфигурация

По умолчанию утилита ищет файл конфигурации .loglint.yaml в корне проекта, где запускается линтер. Если файл не найден, используются стандартные настройки.

    **Важное замечание о приоритетах**
    Проверка чувствительных данных (Sensitive Words) имеет наивысший приоритет. Если в строке обнаружено совпадение, линтер выводит предупреждение о безопасности и прекращает дальнейший анализ этой строки, чтобы сфокусировать внимание разработчика на критической уязвимости.

## Примеры использования
### Проверка проекта

Линтер просканирует все файлы и выведет список нарушений командой:

```bash
./loglint ./...
```

### Автоматическое исправление (-fix)

Линтер может автоматически приводить сообщения к нижнему регистру и скрывать чувствительные данные. Из-за особенностей механизма SuggestedFix, для применения всех исправлений может потребоваться повторный запуск:

```bash
./loglint -fix ./...   # Первый проход

./loglint -fix ./...   # Повторный запуск
```

### Указание пути к конфигурации

Если файл настроек лежит в другом месте, используйте флаг -config:

```bash
./loglint -config=my_config.yaml ./...
```

## Пример работы

Исходный код (debug.go):

```go
log.Print("Starting server!!!") // Ошибка: регистр и спецсимволы
log.Print("password: 12345")    // Ошибка: приоритетное сообщение о безопасности
log.Print(":=")                 // OK (добавлено в allowed_symbols)
```

Вывод линтера:

```text
debug.go:6:12: log message must start with a lowercase letter
debug.go:6:12: log message must not contain special characters
debug.go:7:12: log message contains potentially sensitive data
```

## Разработка и тестирование

Для запуска тестов используйте команду:

```bash
go test -v ./...
```
